# Natural Language And Automation

- Canonical IDs: `F4.2`, `F4.3`
- Status: future
- Summary or outcome: let users describe limited multi-step work in plain language and run it through a visible, controllable workflow engine.

## Scope

- `F4.2` Interpret constrained natural-language requests, map them to supported commands and parameters, and show a proposed plan before execution.
- `F4.3` Add a workflow step format, ordered execution, retries, checkpoints, and confirmation before destructive steps.

## Acceptance Criteria

- A limited set of multi-step requests can be interpreted and executed reliably.
- A multi-step workflow can run from one trigger with visible per-step status.
